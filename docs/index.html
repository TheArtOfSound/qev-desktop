Lock proof into a file. Verify it later.

QEV creates encrypted, tamper-evident vaults for text, AI outputs, logs, research artifacts, notes, and sensitive records.

No account required. No cloud required. No custom crypto claims. Install the CLI, create a vault, and verify it later with the same phrase.

Install npm CLI Read threat model Vault format
$ npm install -g @bryan237l/qev-cli $ qev self-test qev self-test: encrypt → decrypt → tamper → wrong-phrase ... ok $ echo "important proof" | qev lock --out proof.vault $ qev unlock proof.vault

No account

Vaults are local files. The CLI does not need a hosted account model.

Tamper evident

Protected metadata and ciphertext are authenticated. Changed vaults fail to open.

npm available

Install globally or run with npx. The package exposes CLI and programmatic APIs.

Technical model

QEV uses established cryptographic primitives and documents its limits. It is not marketed as a new encryption algorithm.

Best use

Small artifacts: notes, outputs, logs, receipts, and records that need a portable offline envelope.
