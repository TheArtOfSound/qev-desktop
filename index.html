What QEV is.

A narrow tool with an honest security boundary: encrypt one thing, keep the phrase, verify later.

Private notes

Write or paste a sensitive record and export a portable encrypted envelope.

AI receipts

Lock model outputs, prompts, research notes, or timestamped artifacts into a tamper-evident file.

Ops logs

Use the CLI from a terminal or script when you need a small offline vault workflow.

How it works.

QEV is not new cryptography. It packages established primitives into a local workflow.

Write or pipe a record.Use the web vault or CLI. The CLI reads plaintext from stdin.
Choose a phrase.The CLI refuses phrase arguments so your phrase does not leak through shell history.
Export a vault.The vault uses XChaCha20-Poly1305, Argon2id, and deterministic metadata binding.
Verify later.Unlock with the phrase. Tampering with bound fields should fail authentication.

Honest scope.

Security tools should say what they do not do.

QEV does not protect against a compromised device, weak phrase, forgotten phrase, malicious browser extension, or supply-chain compromise.

It is not a password manager, messenger, cloud storage encryption service, or a substitute for a professional security audit. Read the threat model before relying on it for sensitive material.
