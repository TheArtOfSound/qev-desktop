Vault details

1. Lock

Enter text or choose a file. Select strength. Enter a phrase twice. Create and download a .vault file.

2. Verify

Inspect metadata and use test unlock to confirm the phrase works before you trust the vault.

3. Unlock

Upload or paste a vault, enter the phrase, then copy or download the plaintext/file payload.

Honest scope.

This is a usable browser vault. The CLI remains the cleaner high-trust workflow because browser pages can inherit browser, extension, CDN, and supply-chain risks.

QEV does not protect against a compromised device, weak phrase, malicious extension, CDN compromise, or forgotten phrase.

Use the CLI for stronger local control. Use this page for fast public demo, simple vault use, and proof-of-workflow.

Format.

The inline app follows the documented QEV V2 shape: schema, version, created_at, mode, kdf, wrap, and content.

Argon2id

Derives a wrap key from your phrase and random salt using quick, strong, or vault presets.

XChaCha20-Poly1305

Wraps a random content key, then encrypts the payload under that content key.

AAD binding

Schema, version, timestamp, mode, KDF settings, algorithms, and nonces are bound into the seal.
